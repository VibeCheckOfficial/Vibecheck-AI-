## Specification Standards

When writing specifications:

1. **Problem Statement**: Clearly define the problem being solved
2. **Requirements**: List functional and non-functional requirements
3. **Design**: Include diagrams and technical approach
4. **API Surface**: Document public interfaces
5. **Testing Strategy**: Define acceptance criteria
